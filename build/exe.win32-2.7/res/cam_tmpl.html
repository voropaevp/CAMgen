3. Technical Environment

This document provides details of the SYMANTEC data protection solution that is implemented at AMP using NetBackup Enterprise Server. {{ customer.CustomerShortName }} use NBU 7.6.0.1 to protect their mission critical servers, applications and databases.

3.1 Technical Environment

The proposed backup solution for the data center environment consists of the following components:
  • One Microsoft Windows Server 2008 Master Server installed in the primary site
  • The Master Server is clustered using Microsoft Windows Server 2008 MSCS.
  • Tape devices are connected to the master server.
  • Basic disk storage units are connected to the master server.
  • 90 Standard Windows Clients backup across the LAN.
  • NDMP filers backup across the LAN to the tape devices.

5. Basic administrative procedures

This section describes administrative procedures that need to be performed regularly in the NetBackup environment;

Daily Tasks
  1. Open NBU Console and check the Activity Monitor for any failed jobs.
  2. Check the services and drives status.
  3. Verify the volume pools. It's a good practice to ensure volumes are available within the Scratch pool.
  4. Check the volume(s) associated with the catalog and ensure there is enough free space available.

The Activity Monitor topology view displays icons that represent the state and configuration of the entire NetBackup system that is being administered. The Activity Monitor displays only robots and the drives that have storage units configured. If a device host has no configured devices, the device host does not appear in the Activity Monitor. The topology view shows master servers, media servers, clients, and NetBackup storage unit devices.

The Administrators guide should be considered the primary source for NetBackup related administrative procedures. A current copy of the Administrators Guide has been provided with the NBU media kit.

6.2 Symantec Support Phone Number

Symantec Support can be contacted on 800 0441 2244 or local support number.

When calling Symantec Support please ensure that only one of your named contacts makes the call, and that they have the following information available:

  • Description of the problem, including symptoms, exact error messages, scope of users/servers affected, time when problem started, actions taken so far to attempt to resolve the problem. It may also be useful to have traces and screen dumps available.
  • Version of software you are running
  • Version of Operating system, service pack, etc.

You will also be asked to define the severity of the support call:

  • Severity 1 - a "system down" or product inoperative condition that impacts your production / business-critical operation
  • Severity 2 - severely affects or restricts major functionality
  • Severity 3 - issue with no major effect on business systems
  • Severity 4 - minor condition or documentation error

It may expedite your call by making sure you are in front of, or have access to the computer/server you need to troubleshoot.
